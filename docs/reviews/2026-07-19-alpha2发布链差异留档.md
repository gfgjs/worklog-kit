## 公仓 tag 为何错位

公史断档重建(2026-07-19,单基线 `055c405`)时,annotated tag `v0.1.0-alpha.2` 打在了
fresh-export 新基线上,而该基线含**发布之后**的私仓引擎演进(与 registry 差 18 个 package
文件,公仓侧普遍更大)。即 tag 语义从「发布点源码」漂成了「重建日源码」。用户已裁:
公开 tag 不再移动;alpha.2 的对外口径是「tag 与 registry tarball 不同源,以本档为准」。

## 对下一版本的约束(复审 §9 链序,已落 runbook)

1. 版本号只前进(alpha.3+),绝不复用 alpha.2;
2. 净化快照先落公仓,从公仓 exact commit `npm pack` 出唯一 tgz;
3. 对 exact tgz 扫词表/allowlist(sync-public 已内建 tgz 扫描)、装 tgz 跑真 bin e2e
   (`tools/release-selftest.mjs`);
4. publish **该 tgz 本身**(不重复 pack),`npm view` 校 registry shasum;
5. tag 打 exact source commit,复 pack 对 shasum 后才算收口。
